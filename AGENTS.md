# AGENTS.md

Compact guidance for OpenCode sessions working in this repo.

## What this repo is

`create-react-native-init-app` (`rnia`) — an interactive CLI that scaffolds React Native projects from a prebuilt Clean Architecture template. The CLI itself is the product published to npm. It is NOT a React Native app; it is a Node/Bun CLI written in TypeScript with an Ink (React) UI.

The generated project comes from a separate GitHub repo: `alejandro-technology/react-native-template` branch `main` (hardcoded in `src/infrastructure/template/github-template.repository.ts`). This repo does not contain the template source.

## Toolchain

- **Bun is the primary tool.** `bun run dev` runs the CLI directly from TSX source (`src/ui/tui.tsx`). `bun run build` produces the publishable bundle. Do not assume `node`/`tsc` — there is no tsc step.
- Build (`scripts/build.ts`) uses `Bun.build` with `target: "node"`, `minify: true`, single output `bin/index.js`. The build script prepends the `#!/usr/bin/env node` shebang and `chmod 755`s the file.
- **`bin/` is a build artifact and is gitignored.** It is regenerated by `prepublishOnly` before every `npm publish`. Do not edit `bin/index.js` by hand and do not commit it. The published package ships ONLY `bin/` (`package.json#files`).
- Three lockfiles exist (`bun.lockb`, `package-lock.json`, `pnpm-lock.yaml`). Pick Bun for CLI dev work unless explicitly asked otherwise.
- `tsconfig.json` has `strict: false`, `moduleResolution: "bundler"`, `target/module: ESNext`. Source imports use `.js` extensions in relative paths (ESM). Do not drop them.
- Node engine in `package.json` is `>= 22.0.0`; the README says `>= 18` but is stale. Trust `package.json`.

## Architecture (Clean Architecture, 4 layers)

```
src/
  domain/          models, validators, constants, repository interfaces
  application/     use-cases: scaffold (scaffold.use-case.ts), commands/run-command.use-case.ts
  infrastructure/  GithubTemplateRepository (download/extract tar from GitHub)
  ui/              tui.tsx (entrypoint), prompts/ (enquirer), components/ (Ink)
```

Flow: `src/ui/tui.tsx` → `runPrompt()` (enquirer collects user input) → `render(<App>)` (Ink). `App.tsx` dispatches by command:
- `scaffold` → `GithubTemplateRepository.download` then `scaffoldProject` use-case.
- `clean` / `pod-install` / `run-android` → `runCommand` use-case.
- `version` / `help` → rendered inline in `App.tsx`.

## Critical quirks (easy to get wrong)

- **`run-command.use-case.ts` hardcodes `npm run <script>`** for clean/pod-install/run-android, regardless of the package manager the user selected in the prompt. If you "fix" this to respect `PM_COMMANDS`, you are changing behavior — confirm with the user first.
- **Stale constants in `src/domain/constants.ts`.** `VERSION` is `"1.0.1"` but `package.json` is `1.0.10`; `RN_VERSION` is `"0.84.0"` (shown only in `version` output) but `scaffold.use-case.ts` pins `@react-native-community/cli init --version 0.83.4`. When bumping versions, update all three sites deliberately — they do not auto-sync.
- `scaffold.use-case.ts` is the heart of the scaffold logic: it copies files, prunes Firebase modules based on user choices, patches dependency-injection files via hardcoded regex (`replaceInFileIfExists`), and merges `package.json` from the template. Regex patches are line-shape-sensitive — changing formatting in the template (e.g. extra blank lines) will silently break the pruning.
- Agent-config folders (`.claude`, `.opencode`, `.trae`) are copied from a `.ai/` folder in the template, not from a per-agent source. See `CODE_AGENT_FILES` in `scaffold.use-case.ts`.

## Tests

- **There are no tests in this repo.** `npm test` runs `jest` but there is no jest config and no `*.test.ts` files. Do not assume a test suite, fixtures, or coverage gate exists. Verify before claiming "tests pass."
- There is no CI, no husky, no lint/format script, no codegen step in this repo.

## Commands

| Command | What it does (in THIS repo) |
| --- | --- |
| `bun run dev` | Run the CLI interactively from source (TSX). Main loop for manual QA. |
| `bun run build` | Build `bin/index.js` via `scripts/build.ts`. Required before publish. |
| `npm publish` | Publish to npm. `prepublishOnly` runs `bun run build` automatically. |
| `bun run clean` | `rm -rf bin/*` — wipes the BUILD OUTPUT, not caches. (Not the user-facing `clean` command.) |

To exercise the real CLI without `bun run dev`, install built artifact: `node bin/index.js` (or `npm link`).
